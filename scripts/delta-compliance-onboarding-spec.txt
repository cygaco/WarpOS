### GS-ONB-03: Resume Parsing Success

> As a User, I want my uploaded resume parsed into structured fields automatically, so that I see my extracted data without manual entry.

**Depends on:** GS-ONB-01, GS-ONB-06
**Data:** `SessionData.resumeRaw` (read) → `SessionData.personal`, `SessionData.education`, `SessionData.resumeStructured` (write) → `src/lib/types.ts`; PARSE prompt → `src/lib/prompts.ts`

**Verifiable by:** After upload, `SessionData.personal.name` is non-empty; `SessionData.education` is an array; resume parsing was triggered as a background task.

**Acceptance Criteria:**

- Parsed output includes at minimum: name, email, phone, location, education entries, and skills
- Fields not found in the resume are returned as empty rather than hallucinated
- Parsing completes within 30 seconds for files under 10MB
- complete() MUST include ALL produced fields: resumeRaw, personal, education, resumeStructured, context, demographics (HYGIENE Rule 10)
- Note: context and demographics fields are populated with defaults during parsing (empStatus='employed', others=''). Actual user values come from Step 2 preferences (GS-ONB-16, GS-ONB-23).

- Parsing runs as a background task — it does NOT block the user from advancing to preferences
- On completion, results are saved to session silently (no page reload required during preferences)

## 3. Context

Onboarding is the foundation of the entire pipeline. Every downstream feature — market research, resume generation, LinkedIn content, auto-apply — depends on the data collected here. A user who abandons onboarding produces zero value. The flow must feel fast, low-friction, and progressively rewarding: upload a resume, confirm details, set preferences, and receive an AI-generated professional profile — all in under 5 minutes.

## 4. JTBD (Jobs To Be Done)

> When I start using a job search tool, I want to quickly import my resume and set my preferences, so I can get personalized results without manually entering everything.

> When I see my AI-generated profile, I want to verify it understands my strengths and career direction, so I can trust the system to represent me accurately downstream.

> When I have a non-standard background (career gap, industry switch, freelance history), I want the tool to capture that context, so I can get advice that accounts for my real situation, not a generic template.

## 5. Emotional Framing

- **Entry**: Hopeful but skeptical — "Will this actually work for me?" The user has tried job tools before. They need to feel this one is different within 60 seconds of starting.
- **During**: Progressive confidence — each completed sub-step builds trust. Resume parsing should feel like magic ("it got everything right"). Preferences should feel empowering ("I'm in control"). Profile generation should feel validating ("it really understands me").
- **Exit**: Energized and ready — the celebration screen should feel like the starting gun. "My arsenal is being built. Let's go."

## 6. Goals

- User completes full onboarding (resume → preferences → profile) in under 5 minutes
- Resume parsing succeeds on 95%+ of well-formatted resumes (PDF, DOCX, TXT)
- Zero data loss — every field the user fills is persisted to encrypted session
- Profile generation produces actionable output (discipline, seniority, skills, gaps) on first attempt for 90%+ of users
- Preferences capture enough signal to generate relevant search queries downstream

## 7. Assumptions

- Users have a resume in a common format (PDF, DOCX, TXT, or plain text they can paste)
- Claude PARSE prompt handles 95%+ of well-formatted resumes without hallucinating fields
- 10MB file limit is sufficient for any reasonable resume (even image-heavy PDFs)
- Employment status (employed/left/outdated) provides enough context for downstream prompts
- Demographics are optional — the system works without them but produces better results with them
- Bidirectional hourly ↔ salary conversion at 2080 hours/year is a reasonable standard assumption
- **v3:** PARSE and PROFILE are anonymous endpoints (no user JWT required); Cloudflare Turnstile on first anonymous hit adds <100 ms transparent latency (invisible challenge). This upgrades SECURITY.md's v2 "accepted risk" to MVP hardening per research F10 / backend PRD §16 decision 17.
- **v3:** Prompt Caching on PARSE and PROFILE (system prompt + canonical context cached) — mandatory per backend PRD §8.16.

## 8. Feature Description

Onboarding is a 3-step wizard that transforms a raw resume into a structured professional profile with search preferences.

**Step 1: Resume Upload (IntroScreen)**

The user provides their resume via the IntroScreen landing page: drag-and-drop file upload (PDF, DOCX, TXT, MD — 10MB limit) or by pasting raw text. Upon upload, the system immediately triggers resume parsing as a **background task** via Claude AI (PARSE prompt) and advances the user to Step 2 (Preferences). The user does NOT wait for parsing to complete.

**v3 anonymous-endpoint hardening:** PARSE is called via `POST ${API_BASE_URL}/claude` without an auth cookie (anonymous path). On the first call from a fresh browser session, Cloudflare Turnstile issues an invisible challenge token that the frontend attaches as `CF-Turnstile-Response` header on subsequent calls. Backend validates via Siteverify API. Transparent to the user in the normal case; blocks programmatic scanners and low-effort botnets. Request per-ASN rate limit (Cloudflare) further dampens distributed abuse. Per backend PRD §8.2 layer 2 + §16 decision 17.

A persistent parsing status banner is displayed throughout Step 2:

- **During parsing**: "Parsing your resume in the background..." (yellow/olive)
- **On success**: "Resume parsed ✓" (green) — data saved to session silently
- **On failure**: "Resume parsing failed. Retry?" (red) — retry action available; does NOT block preference entry

> **Parse-failed entry state:** If resume parsing fails (timeout, unsupported format), the user sees an error toast and can retry the upload. The system does NOT block advancement to Step 2 -- preferences can be entered manually without parsed data. Only Step 3 (profile generation) gates on parsed resume data.

Session auto-saves at each sub-step boundary. If the user refreshes or returns, they resume where they left off within onboarding. If the user returns with `resumeRaw` in session but no `resumeStructured`, the system auto-triggers parsing without requiring re-upload.

> **Parallel Parse Principle**: Resume parsing and preference collection run concurrently. The user never waits idle for parsing. Questions that don't need resume data (all of Step 2) are asked while the parse runs. Only Step 3 (profile generation) requires parsed data, creating a natural synchronization point.

**Step 2: Preferences & Parameters**

All 6 substeps are **resume-independent** — they collect user intent, not resume data. No substep reads from `resumeStructured`, `personal`, or `education`. Resume parsing may complete during any substep — the banner updates but the flow is uninterrupted.

Step 2 is handled by `OnboardingPage.tsx` as 6 internal substeps: **direction** → **worktype** → **comp** → **location** → **quickcheck** → **dealbreakers**. Each substep auto-saves to session on completion.

Direction: multiselect cards, 1+ required (same/similar role, higher-level, new career, leadership, startup, something else). Default: "Same or similar role". Selecting "new career" or "something else" reveals a free-text detail field.

Work type: 3 single-select pill buttons (Full-time, Part-time, Contract). Default: Full-time. Start date: pill buttons (Immediately, 2 weeks notice, 1 month notice, 2 months notice). Default: Immediately.

Compensation: structure dropdown (default "Open to discussion"). Hourly floor and salary floor text inputs with bidirectional sync (hourly × 2080 = annual).

Location: single-select pill toggle (Remote, Hybrid, On-site). Default: Remote. Non-remote reveals city autocomplete.

Quick check: 7 tappable card-style rows (full-width boxes with checkbox indicator: filled ✓ when active, empty border when inactive). Smart defaults: US citizen ✓, authorized to work ✓, no visa sponsorship ✓, driver's license ✓, veteran ✗, disability ✗, security clearance ✗. User taps any card to toggle. **NOT toggle switches** — use card boxes matching the direction/worktype button pattern.

Deal-breakers: 5 tappable card-style rows (full-width boxes with ✕ indicator when active, empty border when inactive). ALL pre-selected by default (unpaid work/spec work, no benefits offered, return-to-office mandate, requires relocation, travel required). "Return-to-office mandate" hidden if user selected On-site. User taps to deselect. The ✕ indicator (not ✓) reinforces that these are things being rejected/skipped. **NOT toggle switches** — use card boxes matching the quick check pattern.

**Step 3: Profile Generation & Resume Verification**

**Background Profile Generation**: Profile generation (PROFILE Claude call) is triggered immediately after resume parsing completes, rather than waiting for the user to reach Step 3. OnboardingPage orchestrates this: as soon as `triggerBackgroundParse()` succeeds, `triggerBackgroundProfileGen()` fires using whatever context and preferences exist at that moment (defaults for any fields the user hasn't filled yet). A phase ticker runs in OnboardingPage (4 phases x 5 seconds), and status is passed down to Step4Profile via `backgroundProfile`, `backgroundGenStatus`, and `backgroundGenPhase` props.

This means the profile may already be complete by the time the user finishes preferences and arrives at Step 3. The parallel architecture — resume parse, profile generation, and preference collection all overlapping — minimizes idle wait time.

- If profile generation completed during Step 2: the user sees their profile immediately on arrival
- If profile is still generating: the progress bar picks up from the current phase (no restart)
- If parse is still running: show "Waiting for resume parsing to complete..." with progress
- If parse failed: show retry option before profile can generate
- "Regenerate" button re-runs profile generation with the user's final (possibly updated) preferences

A confirmation screen presents personal info and education extracted from the resume — the user reviews and corrects parsed data here (name, email, phone, location, education entries). Required fields: name, email, phone, location. Optional: LinkedIn URL, portfolio URL (validated for HTTP/HTTPS). Education entries are editable inline (add/remove/modify). There is no standalone resume review step — verification happens alongside profile review.

**Late-arriving parse data**: Step4Profile has sync effects that update local `personal` and `education` state when props change (parse results arriving after component mount). This fixes a race condition where rushing through preferences could leave the confirm screen with empty personal fields — the data fills in as soon as parsing completes, even if the component is already mounted.

~~"Edit resume text" was removed in Run 007~~ — users can "Upload different" to start with a new file if needed. The confirm screen focuses on correcting parsed data, not re-editing raw text.

Claude AI (PROFILE prompt) analyzes the resume, context, and preferences to generate a structured professional profile: discipline, seniority level, hard skills list, differentiators, achievements, target direction, gaps to strengthen, avoid terms, and professional domains.

Profile fields stream in progressively as Claude generates them, reducing perceived wait time. A progress indicator shows 4 phases: reading resume, identifying strengths, mapping experience, building profile.

The user can edit discipline and seniority via dropdowns. Target direction is editable inline. A "Regenerate" button re-runs the full profile generation. "Looks good" confirms and advances.

**Post-Onboarding Celebration**

After completing all onboarding steps (target state: resume → preferences → profile → search → analysis; deep dive removed), a celebration screen plays before advancing to the dashboard. This marks the transition from setup to active job hunting. Deep-Dive Q&A is then available as an optional tier-jump activity from the dashboard. The auth gate appears at dashboard entry for unauthenticated users.

## 16. Decisions

1. **Resume re-upload**: "Upload different" button on confirm screen. ~~"Edit resume text" removed in Run 007~~ — re-editing raw text added complexity without proportional value. Users who need changes should upload a new file.
2. **Profile streaming**: Chunked JSON for MVP (parse partial responses client-side). Upgrade to SSE post-MVP.
3. **Demographics skip**: Soft nudge ("Completing demographics improves your results") but allow skip. Downstream features work without demographics but produce better results with them.
4. **Background parse architecture**: Resume parsing runs in background during preference collection. Profile generation also runs in background, triggered immediately after parse completes — it does not wait for the user to reach Step 3. This eliminates 10-30s idle wait and reduces abandonment. Trade-off: resume verification is deferred to the profile review screen (Step 3) rather than being a standalone step between upload and preferences. The background profile uses default preferences for any fields not yet filled; "Regenerate" re-runs with final values.

### Section 3: AI-Generated Profile

Generated from resume + preferences via Claude. All fields pre-filled by AI.

#### Editable fields

| Field | Control | Notes |
|---|---|---|

These are AI-extracted. User curates by de-selecting irrelevant items. No "add new" — if it's not on the resume, it shouldn't be here.

| Field | Chip style | Notes |
|---|---|---|

function DirectionStep({
  ctx,
  onNext,
  onBack,
}: {
  ctx: Partial<Context>;
  onNext: (ctx: Partial<Context>) => void;
  onBack?: () => void;
}) {
  const [selected, setSelected] = useState<string[]>(
    ctx.directionPresets?.length ? ctx.directionPresets : ["same"],
  );
  const [detail, setDetail] = useState(ctx.careerDirectionText ?? "");
  const [detailError, setDetailError] = useState("");

  const needsDetail = selected.some(
    (id) => DIRECTION_PRESETS.find((p) => p.id === id)?.needsDetail,
  );

  function togglePreset(id: string) {
    setSelected((prev) =>
      prev.includes(id) ? prev.filter((x) => x !== id) : [...prev, id],
    );
    setDetailError("");
  }

  function handleNext() {
    if (!selected.length) return;
    if (needsDetail && detail.trim().length < 10) {
      setDetailError(
        "Please describe your direction (at least 10 characters).",
      );

// ── Compensation sub-step ─────────────────────────────────────────────────────

function CompStep({
  prefs,
  onNext,
  onBack,
}: {
  prefs: Partial<Preferences>;
  onNext: (prefs: Partial<Preferences>) => void;
  onBack: () => void;
}) {
  const [compStructure, setCompStructure] = useState(
    prefs.compStructure ?? "Open to discussion",
  );
  const [hourly, setHourly] = useState(prefs.hourlyFloor ?? "");
  const [salary, setSalary] = useState(prefs.salaryFloor ?? "");
  const [errors, setErrors] = useState<{ hourly?: string; salary?: string }>(
    {},
  );

  const empTypes = prefs.employmentTypes ?? ["Full-time"];
  const hasContract =
    empTypes.includes("Contract") || empTypes.includes("Part-time");
  const hasFT = empTypes.includes("Full-time");
  const showBoth = hasContract && hasFT;
  const showHourlyPrimary = hasContract && !hasFT;

  // Bidirectional sync: hourly × 2080 = annual
  function onHourlyChange(val: string) {
    setHourly(val);
    setErrors((e) => ({ ...e, hourly: undefined }));
    const n = parseFloat(val);
    if (!isNaN(n) && n > 0) {
      setSalary(String(Math.round(n * 2080)));
    } else if (val === "") {
      setSalary("");
    }
  }

  function onSalaryChange(val: string) {
    setSalary(val);
    setErrors((e) => ({ ...e, salary: undefined }));
    const n = parseFloat(val.replace(/[,$]/g, ""));
    if (!isNaN(n) && n > 0) {
      setHourly(String(Math.round((n / 2080) * 100) / 100));
    } else if (val === "") {
      setHourly("");
    }
  }

  function validate(): boolean {
    const newErrors: { hourly?: string; salary?: string } = {};
    // BUG-019: zero/negative compensation blocked
    if (hourly !== "") {
      const n = parseFloat(hourly);
      if (isNaN(n) || n <= 0) {
        newErrors.hourly = "Hourly rate must be a positive number.";
      }
    }
    if (salary !== "") {
      const n = parseFloat(salary.replace(/[,$]/g, ""));
      if (isNaN(n) || n <= 0) {
        newErrors.salary = "Annual salary must be a positive number.";
      }
    }
    setErrors(newErrors);
    return Object.keys(newErrors).length === 0;
  }

  function handleNext() {
    if (!validate()) return;
    onNext({
      compStructure,
      hourlyFloor: hourly,
      salaryFloor: salary,
    });
  }

  return (
    <div style={cardStyle}>
      <div>
        <h2 style={h2Style}>What are your compensation expectations?</h2>
      </div>

      <Sel
        label="Compensation structure"
        value={compStructure}
        onChange={setCompStructure}
        options={COMP_STRUCTURES.map((s) => ({ value: s, label: s }))}
      />

      {/* Adaptive compensation inputs */}
      {(showHourlyPrimary || showBoth) && (
        <Inp
          label={showBoth ? "Hourly floor" : "Hourly floor"}
          value={hourly}
          onChange={onHourlyChange}
          placeholder="e.g. 75"
          type="number"
          error={errors.hourly}
        />
      )}

      {(hasFT || showBoth) && (
        <Inp
          label={showBoth ? "Annual salary floor" : "Annual salary floor"}
          value={salary}
          onChange={onSalaryChange}
          placeholder="e.g. $120,000"
          error={errors.salary}
        />
      )}

      {/* Bidirectional sync helper */}
      {hourly && salary && (
        <p style={{ fontSize: 12, color: "var(--text-muted)", margin: 0 }}>
          ${hourly}/hr ↔ ${parseInt(salary).toLocaleString()}/yr (based on 2,080
          hrs/yr)
        </p>
      )}

      <div style={navRowStyle}>
        <Btn variant="ghost" onClick={onBack} aria-label="Go back">
          ← Back
        </Btn>
        <Btn
          variant="primary"
          onClick={handleNext}
          aria-label="Continue to location"
        >
          Next →
        </Btn>
      </div>
    </div>
  );
}

// ── Step3Preferences (orchestrator) ──────────────────────────────────────────

export function Step3Preferences({
  initialSubStep,
  initialPreferences,
  initialContext,
  initialDemographics,
  parseStatus = "idle",
  onRetryParse,
  onSubStepComplete,
  onComplete,
}: Step3PreferencesProps) {
  const [currentSubStep, setCurrentSubStep] = useState<Step3SubStep>(
    initialSubStep ?? "direction",
  );
  const [prefs, setPrefs] = useState<Partial<Preferences>>(
    initialPreferences ?? {},
  );
  const [ctx, setCtx] = useState<Partial<Context>>(initialContext ?? {});
  const [demo, setDemo] = useState<Partial<Demographics>>(
    initialDemographics ?? {},
  );

  const subStepIdx = SUBSTEP_ORDER.indexOf(currentSubStep);

  function goBack() {
    const prev = SUBSTEP_ORDER[subStepIdx - 1];
    if (prev) setCurrentSubStep(prev);
  }

  // BUG-018 FIX: auto-save at each substep boundary via onSubStepComplete
  function advanceWithPrefs(newPrefs: Partial<Preferences>) {
    const merged = { ...prefs, ...newPrefs };
    setPrefs(merged);
    const next = SUBSTEP_ORDER[subStepIdx + 1];
    if (next) {
      onSubStepComplete(currentSubStep, {
        preferences: merged,
        context: ctx,
        demographics: demo,
      });
      setCurrentSubStep(next);
    } else {
      onComplete({ preferences: merged, context: ctx, demographics: demo });
    }
  }

  function advanceWithCtx(newCtx: Partial<Context>) {
    const merged = { ...ctx, ...newCtx };
    setCtx(merged);
    const next = SUBSTEP_ORDER[subStepIdx + 1];
    if (next) {
      onSubStepComplete(currentSubStep, {
        preferences: prefs,
        context: merged,
        demographics: demo,
      });
      setCurrentSubStep(next);
    }
  }

  function advanceWithDemo(newDemo: Partial<Demographics>) {
    const merged = { ...demo, ...newDemo };
    setDemo(merged);
    const next = SUBSTEP_ORDER[subStepIdx + 1];
    if (next) {
      onSubStepComplete(currentSubStep, {
        preferences: prefs,
        context: ctx,
        demographics: merged,
      });
      setCurrentSubStep(next);
    } else {
      onComplete({ preferences: prefs, context: ctx, demographics: merged });
    }
  }

  return (
    <div style={pageStyle}>
      <div style={{ ...cardStyle }}>
        {/* Parse status banner — visible on all substeps */}
        {parseStatus !== "idle" && (
          <ParseBanner status={parseStatus} onRetry={onRetryParse} />
        )}

        {/* Substep content */}
        {currentSubStep === "direction" && (
          <DirectionStep
            ctx={ctx}
            onNext={advanceWithCtx}
            onBack={subStepIdx > 0 ? goBack : undefined}
          />
        )}

        {currentSubStep === "worktype" && (
          <WorkTypeStep
            prefs={prefs}
            onNext={advanceWithPrefs}
            onBack={goBack}
          />
        )}

        {currentSubStep === "comp" && (
          <CompStep prefs={prefs} onNext={advanceWithPrefs} onBack={goBack} />
        )}

        {currentSubStep === "location" && (
          <LocationStep
            prefs={prefs}
            onNext={advanceWithPrefs}
            onBack={goBack}
          />
        )}

        {currentSubStep === "quickcheck" && (
          <QuickCheckStep
            demo={demo}
            onNext={advanceWithDemo}
            onBack={goBack}
          />
        )}

        {currentSubStep === "dealbreakers" && (
          <DealBreakersStep
            prefs={prefs}
            locationTypes={prefs.locationTypes ?? ["Remote"]}
            onNext={(newPrefs) => {
              const merged = { ...prefs, ...newPrefs };
              setPrefs(merged);
              onComplete({
                preferences: merged,
                context: ctx,
                demographics: demo,
              });
            }}
            onBack={goBack}
          />
        )}
      </div>
    </div>
  );
}

        value={entry.degree}

        options={DEGREE_OPTIONS.map((d) => ({ value: d, label: d }))}
        placeholder="Select degree"
        error={errors.degree}
      />
      <Inp
        label="Field of study"
        value={entry.field}
        onChange={(v) => setField("field", v)}
        placeholder="e.g. Computer Science"
        required
        error={errors.field}
      />
      <Inp
        label="School"
        value={entry.school}
        onChange={(v) => setField("school", v)}
        placeholder="e.g. MIT"
        required
        error={errors.school}
      />
      <Sel
        label="Graduation year"
        value={entry.graduation_date}
        onChange={(v) => setField("graduation_date", v)}
        options={GRAD_YEARS.map((y) => ({ value: y, label: y }))}
        placeholder="Year"
      />
      <Inp
        label="GPA (optional)"
        value={entry.gpa}
        onChange={(v) => setField("gpa", v)}
        placeholder="e.g. 3.8"
      />
      <Sel
        label="Honors (optional)"
        value={entry.honors || "N/A"}
        onChange={(v) => setField("honors", v === "N/A" ? "" : v)}
        options={HONORS_OPTIONS.map((h) => ({ value: h, label: h }))}
      />
      <Inp
        label="Notable (optional)"
        value={entry.notable}
        onChange={(v) => setField("notable", v)}
        placeholder="e.g. Published research, thesis title"
      />
      <div style={{ display: "flex", gap: 8, justifyContent: "flex-end" }}>
        <Btn variant="ghost" size="sm" onClick={onCancel}>
          Cancel
        </Btn>
        <Btn variant="primary" size="sm" onClick={handleSave}>
          Save
        </Btn>
      </div>
    </div>
  );
}

// ── Chip (de-selectable) ──────────────────────────────────────────────────────

function Chip({
  label,
  active,
  onToggle,
}: {
  label: string;
  active: boolean;
  onToggle: () => void;
}) {
  return (
    <Btn
      variant="chip"
      size="sm"
      selected={active}
      onClick={onToggle}
      aria-pressed={active}
      style={{
        borderRadius: "var(--radius-full)",
        background: active ? "var(--surface-alt)" : "transparent",
        color: active ? "var(--text)" : "var(--text-muted)",
        border: `1px solid ${active ? "var(--border)" : "transparent"}`,
        opacity: active ? 1 : 0.45,
        textDecoration: active ? "none" : "line-through",
        transition: "all 0.15s",
      }}
    >
      {label}
    </Btn>
  );
}

// ── Section wrapper ───────────────────────────────────────────────────────────

function Section({
  title,
  subtitle,
  children,
}: {
  title: string;
  subtitle?: string;
  children: React.ReactNode;
}) {
  return (
    <div
      style={{
        display: "flex",
        flexDirection: "column",
        gap: 12,
      }}
    >
      <div>
        <h3
          style={{
            fontSize: 16,
            fontWeight: 700,
            color: "var(--text)",
            margin: "0 0 2px 0",
          }}
        >
          {title}
        </h3>
        {subtitle && (
          <p style={{ fontSize: 13, color: "var(--text-muted)", margin: 0 }}>
            {subtitle}
          </p>
        )}
      </div>
      {children}
    </div>
  );
}

// ── Step4Profile ──────────────────────────────────────────────────────────────

export function Step4Profile({
  resumeStructured,
  personal: personalProp,
  education: educationProp,
  context,
  preferences,
  parseStatus,
  existingProfile,
  backgroundProfile,
  backgroundGenStatus = "idle",
  backgroundGenPhase = 0,
  backgroundGenError,
  onRetryParse,
  onComplete,
  onUploadDifferent,
}: Step4ProfileProps) {
  // ── Personal info local state + touched guard (GS-ONB-34) ────────────────
  const [personal, setPersonal] = useState<Personal>(
    personalProp ?? EMPTY_PERSONAL,
  );
  const [personalTouched, setPersonalTouched] = useState(false);

  // ── Education local state + touched guard (GS-ONB-34) ────────────────────
  const [education, setEducation] = useState<EducationEntry[]>(
    educationProp ?? [],
  );
  const [educationTouched, setEducationTouched] = useState(false);
  const [editingEduIdx, setEditingEduIdx] = useState<number | null>(null);
  const [addingEdu, setAddingEdu] = useState(false);

  // ── Profile state ─────────────────────────────────────────────────────────
  const [localProfile, setLocalProfile] = useState<Profile | null>(
    existingProfile ?? backgroundProfile ?? null,
  );
  const [genStatus, setGenStatus] = useState<Step4ProfileGenStatus>(() => {
    if (existingProfile || backgroundProfile) return "done";
    return backgroundGenStatus;
  });
  const [genPhase, setGenPhase] = useState<number>(backgroundGenPhase);
  const [genError, setGenError] = useState<string | null>(
    backgroundGenError ?? null,
  );

  // ── Profile display state (editable fields) ───────────────────────────────
  const [discipline, setDiscipline] = useState(localProfile?.discipline ?? "");
  const [seniority, setSeniority] = useState(localProfile?.seniority ?? "");
  const [targetDirection, setTargetDirection] = useState(
    localProfile?.targetDirection ?? "",
  );
  const [activeSkills, setActiveSkills] = useState<Set<string>>(
    new Set(localProfile?.hardSkills ?? []),
  );
  const [activeDomains, setActiveDomains] = useState<Set<string>>(
    new Set(localProfile?.domains ?? []),
  );

  // ── Validation errors ─────────────────────────────────────────────────────
  const [personalErrors, setPersonalErrors] = useState<Record<string, string>>(
    {},
  );
  const [portfolioError, setPortfolioError] = useState<string | null>(null);

  // ── AbortController for generation ───────────────────────────────────────
  const abortRef = useRef<AbortController | null>(null);
  const phaseTimerRef = useRef<ReturnType<typeof setInterval> | null>(null);

  // ── GS-ONB-34: Sync personal from props when late parse arrives ───────────
  useEffect(() => {
    if (!personalTouched && personalProp?.name) {
      setPersonal(personalProp);
    }
  }, [personalProp]); // eslint-disable-line react-hooks/exhaustive-deps

  // ── GS-ONB-34: Sync education from props when late parse arrives ──────────
  useEffect(() => {
    if (!educationTouched && educationProp?.length) {
      setEducation(educationProp);
    }
  }, [educationProp]); // eslint-disable-line react-hooks/exhaustive-deps

  // ── GS-ONB-33: React to backgroundGenStatus prop changes ─────────────────
  useEffect(() => {
    if (backgroundGenStatus === "done" && backgroundProfile && !localProfile) {
      setLocalProfile(backgroundProfile);
      setGenStatus("done");
      applyProfile(backgroundProfile);
    } else if (backgroundGenStatus === "generating") {
      if (genStatus !== "done") {
        setGenStatus("generating");
        setGenPhase(backgroundGenPhase);
      }
    } else if (backgroundGenStatus === "error" && genStatus !== "done") {
      setGenStatus("error");
      setGenError(backgroundGenError ?? "Profile generation failed.");
    }
  }, [
    backgroundGenStatus,
    backgroundProfile,
    backgroundGenPhase,
    backgroundGenError,
  ]); // eslint-disable-line react-hooks/exhaustive-deps

  // ── Auto-trigger generation if background gen didn't run ─────────────────
  useEffect(() => {
    if (
      genStatus === "idle" &&
      !localProfile &&
      parseStatus === "done" &&
      resumeStructured
    ) {
      triggerGeneration();
    }
  }, []); // eslint-disable-line react-hooks/exhaustive-deps

  // ── Cleanup on unmount ────────────────────────────────────────────────────
  useEffect(() => {
    return () => {
      abortRef.current?.abort();
      if (phaseTimerRef.current) clearInterval(phaseTimerRef.current);
    };
  }, []);

  function applyProfile(p: Profile) {
    setDiscipline(p.discipline ?? "");
    setSeniority(p.seniority ?? "");
    setTargetDirection(p.targetDirection ?? "");
    setActiveSkills(new Set(p.hardSkills ?? []));
    setActiveDomains(new Set(p.domains ?? []));
  }

  function startPhaseTicker() {
    let phase = 0;
    setGenPhase(0);
    if (phaseTimerRef.current) clearInterval(phaseTimerRef.current);
    phaseTimerRef.current = setInterval(() => {
      phase = Math.min(phase + 1, PROFILE_PHASES.length - 1);
      setGenPhase(phase);
    }, 5000);
  }

  function stopPhaseTicker() {
    if (phaseTimerRef.current) {
      clearInterval(phaseTimerRef.current);
      phaseTimerRef.current = null;
    }
  }

  const triggerGeneration = useCallback(async () => {
    if (genStatus === "generating") return;

    // Cancel any in-flight generation
    abortRef.current?.abort();
    abortRef.current = new AbortController();

    setGenStatus("generating");
    setGenError(null);
    startPhaseTicker();

    const profileInput = JSON.stringify({
      resume: resumeStructured,
      personal: personalProp,
      context: {
        empStatus: context.empStatus ?? "employed",
        careerDirection:
          context.careerDirection ?? context.careerDirectionText ?? "",
        directionPresets: context.directionPresets ?? [],
        whyLeft: context.whyLeft ?? "",
        latestRoleText: context.latestRoleText ?? "",
      },
      preferences: {
        locationTypes: preferences.locationTypes ?? ["Remote"],
        employmentTypes: preferences.employmentTypes ?? ["Full-time"],
        compStructure: preferences.compStructure ?? "Open to discussion",
        salaryFloor: preferences.salaryFloor ?? "",
        hourlyFloor: preferences.hourlyFloor ?? "",
        startDate: preferences.startDate ?? "Immediately",
        dealBreakers: preferences.dealBreakers ?? [],
      },
    });

    try {
      const raw = await callClaude(
        "PROFILE",
        profileInput,
        4096,
        abortRef.current.signal,
      );
      stopPhaseTicker();

      const parsed = JSON.parse(cleanJson(raw)) as Profile;
      setLocalProfile(parsed);
      applyProfile(parsed);
      setGenStatus("done");
    } catch (err) {
      stopPhaseTicker();
      if (err instanceof Error && err.message.includes("aborted")) return;
      setGenStatus("error");
      setGenError(
        safeErrorMessage(err, "Profile generation failed. Please try again."),
      );
    }
  }, [resumeStructured, personalProp, context, preferences]); // eslint-disable-line react-hooks/exhaustive-deps

  // ── Personal field updater ────────────────────────────────────────────────
  function updatePersonal(field: keyof Personal, value: string) {
    setPersonalTouched(true);
    setPersonal((prev) => ({ ...prev, [field]: value }));
    setPersonalErrors((prev) => ({ ...prev, [field]: "" }));
    if (field === "portfolio") setPortfolioError(null);
  }

  // ── Confirm + validate ────────────────────────────────────────────────────
  function handleConfirm() {
    if (!localProfile) return;

    const errs: Record<string, string> = {};
    if (!personal.name.trim()) errs.name = "Name is required";
    if (!personal.email.trim()) errs.email = "Email is required";
    if (!personal.phone.trim()) errs.phone = "Phone is required";
    if (!personal.location.trim()) errs.location = "Location is required";

    const portErr = validatePortfolio(personal.portfolio ?? "");
    if (portErr) {
      setPortfolioError(portErr);
      return;
    }

    if (Object.keys(errs).length) {
      setPersonalErrors(errs);
      return;
    }

    // Build final profile with user overrides
    const finalProfile: Profile = {
      ...localProfile,
      discipline,
      seniority,
      targetDirection,
      hardSkills: (localProfile.hardSkills ?? []).filter((s) =>
        activeSkills.has(s),
      ),
      domains: (localProfile.domains ?? []).filter((d) => activeDomains.has(d)),
    };

    // GS-ONB-24: context defaults (no empStatus etc on confirm screen)
    const finalContext: Partial<Context> = {
      ...context,
      empStatus: context.empStatus ?? "employed",
      notOnResume: context.notOnResume ?? "",
      titleNotCapture: context.titleNotCapture ?? "",
    };

    onComplete({
      profile: finalProfile,
      personal,
      education,
      context: finalContext,
    });
  }

  // ── Shared page layout ────────────────────────────────────────────────────
  const outerStyle: React.CSSProperties = {
    display: "flex",
    flexDirection: "column",
    alignItems: "center",
    padding: "32px 24px 80px",
    background: "var(--bg)",
    minHeight: "100%",
  };
  const contentStyle: React.CSSProperties = {
    display: "flex",
    flexDirection: "column",
    width: "100%",
    maxWidth: 600,
    gap: 32,
  };

  // ── GS-ONB-36: Waiting for parse ──────────────────────────────────────────
  if (parseStatus === "parsing") {
    return (
      <div style={outerStyle}>
        <div
          style={{
            ...contentStyle,
            alignItems: "center",
            gap: 20,
            paddingTop: 60,
          }}
        >
          <Spin size="lg" />
          <div style={{ textAlign: "center" }}>
            <h2
              style={{
                fontSize: 20,
                fontWeight: 700,
                color: "var(--text)",
                margin: "0 0 8px 0",
              }}
            >
              Waiting for resume parsing to complete…
            </h2>
            <p style={{ fontSize: 14, color: "var(--text-muted)", margin: 0 }}>
              Profile generation will start as soon as parsing completes.
            </p>
            <div
              style={{
                marginTop: 16,
                display: "flex",
                flexDirection: "column",
                gap: 4,
              }}
            >
              {[
                "Reading document…",
                "Extracting structure…",
                "Building profile…",
              ].map((p) => (
                <p
                  key={p}
                  style={{
                    fontSize: 13,
                    color: "var(--text-muted)",
                    margin: 0,
                  }}
                >
                  {p}
                </p>
              ))}
            </div>
          </div>
        </div>
      </div>
    );
  }

  // ── GS-ONB-37: Parse error recovery ──────────────────────────────────────
  if (parseStatus === "error") {
    return (
      <div style={outerStyle}>
        <div
          style={{
            ...contentStyle,
            alignItems: "center",
            gap: 20,
            paddingTop: 60,
          }}
        >
          <div
            style={{
              background: "var(--error-light)",
              border: "1px solid var(--error-border)",
              borderRadius: "var(--radius-lg)",
              padding: "24px",
              textAlign: "center",
              width: "100%",
            }}
          >
            <p
              style={{
                fontSize: 16,
                fontWeight: 600,
                color: "var(--error)",
                margin: "0 0 8px 0",
              }}
            >
              Resume parsing failed.
            </p>
            <p
              style={{
                fontSize: 14,
                color: "var(--text-muted)",
                margin: "0 0 16px 0",
              }}
            >
              We couldn&apos;t extract your resume data. Try again or upload a
              different file.
            </p>
            <div style={{ display: "flex", gap: 10, justifyContent: "center" }}>
              {onRetryParse && (
                <Btn
                  variant="primary"
                  onClick={onRetryParse}
                  aria-label="Retry resume parsing"
                >
                  Retry
                </Btn>
              )}
              {onUploadDifferent && (
                <Btn
                  variant="ghost"
                  onClick={onUploadDifferent}
                  aria-label="Upload a different resume"
                >
                  Upload different
                </Btn>
              )}
            </div>
          </div>
        </div>
      </div>
    );
  }

  // ── Profile generating ────────────────────────────────────────────────────
  if (genStatus === "generating") {
    return (
      <div style={outerStyle}>
        <div
          style={{
            ...contentStyle,
            alignItems: "center",
            gap: 24,
            paddingTop: 60,
          }}
        >
          <Spin size="lg" />
          <div style={{ textAlign: "center" }}>
            <h2
              style={{
                fontSize: 20,
                fontWeight: 700,
                color: "var(--text)",
                margin: "0 0 16px 0",
              }}
            >
              Building your profile…
            </h2>
            <div style={{ display: "flex", flexDirection: "column", gap: 6 }}>
              {PROFILE_PHASES.map((phase, i) => (
                <p
                  key={phase}
                  style={{
                    fontSize: 14,
                    color:
                      i === genPhase
                        ? "var(--text)"
                        : i < genPhase
                          ? "var(--success)"
                          : "var(--border)",
                    fontWeight: i === genPhase ? 600 : 400,
                    margin: 0,
                    transition: "color 0.3s",
                  }}
                >
                  {i < genPhase ? "✓ " : ""}
                  {phase}
                </p>
              ))}
            </div>
          </div>
        </div>
      </div>
    );
  }

  // ── GS-ONB-38: Generation error ───────────────────────────────────────────
  if (genStatus === "error") {
    return (
      <div style={outerStyle}>
        <div
          style={{
            ...contentStyle,
            alignItems: "center",
            gap: 20,
            paddingTop: 60,
          }}
        >
          <div
            style={{
              background: "var(--error-light)",
              border: "1px solid var(--error-border)",
              borderRadius: "var(--radius-lg)",
              padding: "24px",
              textAlign: "center",
              width: "100%",
            }}
          >
            <p
              style={{
                fontSize: 15,
                color: "var(--error)",
                margin: "0 0 8px 0",
                fontWeight: 600,
              }}
            >
              Profile generation failed.
            </p>
            {genError && (
              <p
                style={{
                  fontSize: 13,
                  color: "var(--text-muted)",
                  margin: "0 0 16px 0",
                }}
              >
                {genError}
              </p>
            )}
            <Btn
              variant="primary"
              onClick={triggerGeneration}
              aria-label="Retry profile generation"
            >
              Retry generation
            </Btn>
          </div>
        </div>
      </div>
    );
  }

  // ── genStatus === 'idle' and no profile yet ───────────────────────────────
  if (genStatus === "idle" && !localProfile) {
    return (
      <div style={outerStyle}>
        <div style={{ ...contentStyle, alignItems: "center", paddingTop: 60 }}>
          <Spin size="lg" />
        </div>
      </div>
    );
  }

  // ── Full review form ──────────────────────────────────────────────────────

  const profile = localProfile!;

  const friendlyName = personal.name ? `, ${personal.name.split(" ")[0]}` : "";

  return (
    <div style={outerStyle}>
      <div style={contentStyle}>
        {/* Friendly intro */}
        <div>
          <h2
            style={{
              fontSize: 22,
              fontWeight: 700,
              color: "var(--text)",
              margin: "0 0 6px 0",
            }}
          >
            Confirm your info
          </h2>
          <p style={{ fontSize: 14, color: "var(--text-muted)", margin: 0 }}>
            Thanks{friendlyName}. Fix anything that&apos;s off — this powers
            everything we build for you.
          </p>
        </div>

        {/* ── Section: Personal ─────────────────────────────────────────── */}
        <Section
          title="Personal"
          subtitle="We extracted this from your resume. Edit anything that looks off."
        >
          <Inp
            label="Name"
            value={personal.name}
            onChange={(v) => updatePersonal("name", v)}
            required
            error={personalErrors.name}
          />
          <Inp
            label="Email"
            value={personal.email}
            onChange={(v) => updatePersonal("email", v)}
            type="email"
            required
            error={personalErrors.email}
          />
          <Inp
            label="Phone"
            value={personal.phone}
            onChange={(v) => updatePersonal("phone", v)}
            required
            error={personalErrors.phone}
          />
          <Inp
            label="Location"
            value={personal.location}
            onChange={(v) => updatePersonal("location", v)}
            required
            error={personalErrors.location}
          />
          <Inp
            label="LinkedIn URL"
            value={personal.linkedin}
            onChange={(v) => updatePersonal("linkedin", v)}
            placeholder="https://linkedin.com/in/yourname"
          />
          <div>
            <Inp
              label="Portfolio URL (optional)"
              value={personal.portfolio ?? ""}
              onChange={(v) => {
                updatePersonal("portfolio", v);
                setPortfolioError(null);
              }}
              placeholder="https://yoursite.com"
              error={portfolioError ?? undefined}
            />
          </div>
        </Section>

        {/* ── Section: Education ────────────────────────────────────────── */}
        <Section title="Education">
          {education.length === 0 && !addingEdu && (
            <p style={{ fontSize: 14, color: "var(--text-muted)" }}>
              No education found on your resume. Want to add one?
            </p>
          )}

          {education.map((edu, idx) =>
            editingEduIdx === idx ? (
              <EduForm
                key={idx}
                initial={edu}
                onSave={(updated) => {
                  setEducationTouched(true);
                  setEducation((prev) =>
                    prev.map((e, i) => (i === idx ? updated : e)),
                  );
                  setEditingEduIdx(null);
                }}
                onCancel={() => setEditingEduIdx(null)}
              />
            ) : (
              <EduCard
                key={idx}
                education={edu}
                onEdit={() => setEditingEduIdx(idx)}
                onRemove={() => {
                  setEducationTouched(true);
                  setEducation((prev) => prev.filter((_, i) => i !== idx));
                }}
              />
            ),
          )}

          {addingEdu && (
            <EduForm
              onSave={(entry) => {
                setEducationTouched(true);
                setEducation((prev) => [...prev, entry]);
                setAddingEdu(false);
              }}
              onCancel={() => setAddingEdu(false)}
            />
          )}

          {!addingEdu && editingEduIdx === null && (
            <Btn
              variant="ghost"
              size="sm"
              onClick={() => setAddingEdu(true)}
              style={{ alignSelf: "flex-start" }}
              aria-label="Add education entry"
            >
              + Add education
            </Btn>
          )}
        </Section>

        {/* ── Section: Who you are (AI Profile) ────────────────────────── */}
        <Section title="Who you are" subtitle="Your professional identity.">
          {/* GS-ONB-38: show error box in profile section if gen failed after partial use */}

          {/* GS-ONB-19: Discipline MUST be free-text Inp, NOT Sel */}
          <Inp
            label="Discipline"
            value={discipline}
            onChange={setDiscipline}
            placeholder="e.g. Product Manager"
          />
          <Sel
            label="Seniority"
            value={seniority}
            onChange={setSeniority}
            options={SENIORITY_OPTIONS.map((s) => ({ value: s, label: s }))}
            placeholder="Select seniority"
          />
        </Section>

        {/* ── Section: Skills ───────────────────────────────────────────── */}
        {(profile.hardSkills?.length ?? 0) > 0 && (
          <Section
            title="Your skills"
            subtitle="We'll use these to match you with jobs."
          >
            <div style={{ display: "flex", flexWrap: "wrap", gap: 6 }}>
              {profile.hardSkills?.map((skill) => (
                <Chip
                  key={skill}
                  label={skill}
                  active={activeSkills.has(skill)}
                  onToggle={() =>
                    setActiveSkills((prev) => {
                      const next = new Set(prev);
                      if (next.has(skill)) next.delete(skill);
                      else next.add(skill);
                      return next;
                    })
                  }
                />
              ))}
            </div>
            <p style={{ fontSize: 12, color: "var(--text-muted)", margin: 0 }}>
              Extracted {activeSkills.size} skills
              {profile.differentiators
                ? ` and ${profile.differentiators.length} differentiators`
                : ""}{" "}
              from your resume. You&apos;ll curate these after the market scan.
            </p>
          </Section>
        )}

        {/* ── Section: Domains ──────────────────────────────────────────── */}
        {(profile.domains?.length ?? 0) > 0 && (
          <Section title="Domains">
            <div style={{ display: "flex", flexWrap: "wrap", gap: 6 }}>
              {profile.domains?.map((d) => (
                <Chip
                  key={d}
                  label={d}
                  active={activeDomains.has(d)}
                  onToggle={() =>
                    setActiveDomains((prev) => {
                      const next = new Set(prev);
                      if (next.has(d)) next.delete(d);
                      else next.add(d);
                      return next;
                    })
                  }
                />
              ))}
            </div>
          </Section>
        )}

        {/* ── Section: What makes you stand out ────────────────────────── */}
        {(profile.differentiators?.length ?? 0) > 0 && (
          <Section
            title="What makes you stand out"
            subtitle="Your competitive edge."
          >
            <ul
              style={{
                margin: 0,

// Step 1: Resume upload  →  Step 2: Preferences (6 substeps)  →  Step 3: Profile review
//
// BUG-017 FIX: initialResumeRaw passed to Step1Resume for returning-user flow.
// BUG-018 FIX: saveSession() at every substep boundary BEFORE advancing (HYGIENE Rule 29).
//
// Architecture:
//   • Background PARSE triggered on upload; user proceeds to Step 2 immediately.
//   • Background PROFILE GEN triggered when parse completes (during Step 2 ideally).
//   • Step4Profile receives bgGenStatus/bgProfile via props; reacts via GS-ONB-33 useEffect.
//   • sessionRef keeps a mutable latest-session copy so async callbacks never read stale state.

import React, { useState, useEffect, useRef } from "react";
import { loadSession, saveSession } from "@/lib/storage";
import { callClaude, cleanJson, safeErrorMessage } from "@/lib/api";

  name: "",
  email: "",
  phone: "",
  location: "",
  linkedin: "",
  portfolio: "",
};

const EMPTY_RESUME_STRUCTURED: ResumeStructured = {
  roles: [],
  education: [],
};

  direction: 0,
  worktype: 1,
  comp: 2,
  location: 3,
  quickcheck: 4,
  dealbreakers: 5,

// ── OnboardingPage ────────────────────────────────────────────────────────────

export function OnboardingPage() {
  // ── Session load state ────────────────────────────────────────────────────
  const [sessionLoaded, setSessionLoaded] = useState(false);
  const [session, setSession] = useState<SessionData>({});
  // Mutable ref — always current; safe to read from async callbacks
  const sessionRef = useRef<SessionData>({});

  // ── Routing ───────────────────────────────────────────────────────────────
  const [innerStep, setInnerStep] = useState<InnerStep>(1);

  // ── Background parse ──────────────────────────────────────────────────────
  const [parseStatus, setParseStatus] = useState<Step4ParseStatus>("idle");
  const [resumeStructured, setResumeStructured] = useState<ResumeStructured>(
    EMPTY_RESUME_STRUCTURED,
  );
  const parseAbortRef = useRef<AbortController | null>(null);

  // ── Background profile generation ─────────────────────────────────────────
  const [bgGenStatus, setBgGenStatus] = useState<Step4ProfileGenStatus>("idle");
  const [bgGenPhase, setBgGenPhase] = useState(0);
  const [bgGenError, setBgGenError] = useState<string | null>(null);
  const [bgProfile, setBgProfile] = useState<Profile | null>(null);
  const profileGenAbortRef = useRef<AbortController | null>(null);
  const phaseTickerRef = useRef<ReturnType<typeof setInterval> | null>(null);

  // ── Preferences substep progress ─────────────────────────────────────────
  // Tracks which dot is "active" in OnboardingProgress
  const [prefSubStep, setPrefSubStep] = useState(0);

  // ── Background profile generation ────────────────────────────────────────

  async function triggerProfileGen(parsedResume: ResumeStructured) {
    // Cancel any in-flight gen
    profileGenAbortRef.current?.abort();
    const controller = new AbortController();
    profileGenAbortRef.current = controller;

    // Clear phase ticker
    if (phaseTickerRef.current) {
      clearInterval(phaseTickerRef.current);
      phaseTickerRef.current = null;
    }

    setBgGenStatus("generating");
    setBgGenPhase(0);
    setBgProfile(null);
    setBgGenError(null);

    // Phase ticker — advances every 5s through 4 phases
    let phase = 0;
    phaseTickerRef.current = setInterval(() => {
      phase = Math.min(phase + 1, 3);
      setBgGenPhase(phase);
    }, 5000);

    // Use sessionRef.current for preferences/context — by the time gen starts,
    // the user may have already completed step 2 (full data available).
    const profileInput = JSON.stringify({
      resume: parsedResume,
      personal: parsedResume.personal ?? EMPTY_PERSONAL,
      context: sessionRef.current.context ?? {},
      preferences: sessionRef.current.preferences ?? {},
    });

    try {
      const raw = await callClaude(
        "PROFILE",
        profileInput,
        4096,
        controller.signal,
      );
      if (controller.signal.aborted) return;

      const cleaned = cleanJson(raw);
      const profile = JSON.parse(cleaned) as Profile;

      if (phaseTickerRef.current) {
        clearInterval(phaseTickerRef.current);
        phaseTickerRef.current = null;
      }

      setBgProfile(profile);
      setBgGenStatus("done");

      // Persist to session (non-blocking — don't await, not critical path)
      const newSession: SessionData = { ...sessionRef.current, profile };
      sessionRef.current = newSession;
      setSession(newSession);
      saveSession(newSession).catch(() => {});
    } catch (err) {
      if (controller.signal.aborted) return;

      if (phaseTickerRef.current) {
        clearInterval(phaseTickerRef.current);
        phaseTickerRef.current = null;
      }

      setBgGenStatus("error");
      setBgGenError(safeErrorMessage(err, "Profile generation failed."));
    }
  }

  // ── Background parse ──────────────────────────────────────────────────────

  async function triggerParse(resumeRaw: string) {
    // Cancel any in-flight parse
    parseAbortRef.current?.abort();
    const controller = new AbortController();
    parseAbortRef.current = controller;

    setParseStatus("parsing");

    try {
      const raw = await callClaude("PARSE", resumeRaw, 4096, controller.signal);
      if (controller.signal.aborted) return;

      const cleaned = cleanJson(raw);
      const parsed = JSON.parse(cleaned) as ResumeStructured;

      setResumeStructured(parsed);
      setParseStatus("done");

      // Merge parsed data into session
      const updated: SessionData = {
        ...sessionRef.current,
        resumeStructured: parsed,
        // Carry personal + education extracted from parse (only if not already user-set)
        personal: sessionRef.current.personal ?? parsed.personal,
        education: sessionRef.current.education?.length
          ? sessionRef.current.education
          : (parsed.education ?? []),
      };
      sessionRef.current = updated;
      setSession(updated);
      await saveSession(updated);

      // Immediately kick off profile gen (runs in background while user is on step 2 or 3)
      triggerProfileGen(parsed);
    } catch (err) {
      if (controller.signal.aborted) return;
      setParseStatus("error");
      // Error is surfaced via ParseBanner — no need to store separately
      void safeErrorMessage(err);
    }
  }

  // ── Mount: load session and route to correct step ─────────────────────────

  useEffect(() => {
    loadSession().then((s) => {
      const loaded = s ?? {};
      sessionRef.current = loaded;
      setSession(loaded);

      const step = loaded.currentStep ?? 1;

      if (step >= 3) {
        // User is at step 3 (profile) — already has parsed resume
        setInnerStep(3);
        if (loaded.resumeStructured) {
          setResumeStructured(loaded.resumeStructured);
        }
        // If bg gen was already persisted (previous session), treat as done
        // Step4Profile will receive existingProfile and skip re-gen
        setParseStatus("done");
      } else if (step === 2) {
        // User is at step 2 (preferences)
        setInnerStep(2);
        if (loaded.resumeStructured) {
          // Parse already done in a previous session
          setResumeStructured(loaded.resumeStructured);
          setParseStatus("done");
          // Re-trigger profile gen if it wasn't completed yet
          if (!loaded.profile) {
            triggerProfileGen(loaded.resumeStructured);
          }
        } else if (loaded.resumeRaw) {
          // Parse didn't complete — re-trigger it
          triggerParse(loaded.resumeRaw);
        }
      } else {
        // Default to step 1
        setInnerStep(1);
      }

      setSessionLoaded(true);
    });

    return () => {
      // Cleanup all in-flight async work on unmount
      parseAbortRef.current?.abort();
      profileGenAbortRef.current?.abort();
      if (phaseTickerRef.current) clearInterval(phaseTickerRef.current);
    };
  }, []); // eslint-disable-line react-hooks/exhaustive-deps

  // ── Step 1: Upload handler ─────────────────────────────────────────────────

  async function handleUpload(resumeRaw: string) {

    const newSession: SessionData = {
      ...sessionRef.current,

      currentStep: 2,
      maxStep: Math.max(sessionRef.current.maxStep ?? 1, 2),

    setSession(newSession);
    await saveSession(newSession);

    setInnerStep(2);
    setPrefSubStep(0);

    // Kick off background parse
    triggerParse(resumeRaw);
  }

  // ── Step 2: Substep complete (BUG-018: save before advancing) ─────────────

// src/lib/upload.ts — Client-side resume file extraction helpers
// BUG-020 FIX: DOCX extraction uses JSZip for proper deflate decompression.

export type ValidateResult = { valid: true } | { valid: false; error: string };

// ── Constants (from VALIDATION_RULES.md) ─────────────────────────────────────

const MAX_FILE_SIZE = 10 * 1024 * 1024; // 10 MB
const MAX_PDF_PAGES = 50;
const MAX_EXTRACTED_TEXT = 500_000; // chars
const PARSE_TIMEOUT_MS = 15_000;

const ALLOWED_EXTENSIONS = new Set([".pdf", ".docx", ".txt", ".md"]);

function getExtension(filename: string): string {
  const lastDot = filename.lastIndexOf(".");
  if (lastDot === -1) return "";
  return filename.slice(lastDot).toLowerCase();
}

// ── validateFile (synchronous — used by page.tsx and components) ──────────────

export function validateFile(file: File): ValidateResult {
  if (file.size === 0) {
    return { valid: false, error: "File is empty." };
  }
  const sizeMb = (file.size / (1024 * 1024)).toFixed(1);
  if (file.size > MAX_FILE_SIZE) {
    return {
      valid: false,
      error: `File too large (${sizeMb}MB). Maximum is 10MB.`,
    };
  }
  const ext = getExtension(file.name);
  if (!ALLOWED_EXTENSIONS.has(ext)) {
    return {
      valid: false,
      error: "Unsupported file type. Accepted: PDF, DOCX, TXT, MD.",
    };
  }
  return { valid: true };
}

// ── validateFileDeep (async — adds magic byte check) ─────────────────────────

export async function validateFileDeep(file: File): Promise<ValidateResult> {
  const basic = validateFile(file);
  if (!basic.valid) return basic;

  const ext = getExtension(file.name);

  // Read first 8 bytes for magic byte header check
  const slice = file.slice(0, 8);
  const buffer = await slice.arrayBuffer();
  const bytes = new Uint8Array(buffer);

  if (ext === ".pdf") {
    // %PDF = 0x25 0x50 0x44 0x46
    if (
      bytes[0] !== 0x25 ||
      bytes[1] !== 0x50 ||
      bytes[2] !== 0x44 ||
      bytes[3] !== 0x46
    ) {
      return { valid: false, error: "File does not appear to be a valid PDF." };
    }
  } else if (ext === ".docx") {
    // PK = 0x50 0x4B 0x03 0x04 (ZIP local file header)
    if (
      bytes[0] !== 0x50 ||
      bytes[1] !== 0x4b ||
      bytes[2] !== 0x03 ||
      bytes[3] !== 0x04
    ) {
      return {
        valid: false,
        error: "File does not appear to be a valid DOCX.",
      };
    }
  }

  return { valid: true };
}

// ── extractText (deep-validate + parse, with timeout) ────────────────────────

export async function extractText(file: File): Promise<string> {
  const check = await validateFileDeep(file);
  if (!check.valid) throw new Error(check.error);

  const ext = getExtension(file.name);

  let extractPromise: Promise<string>;
  if (ext === ".pdf") {
    extractPromise = extractPdfText(file);
  } else if (ext === ".docx") {
    extractPromise = extractDocxText(file);
  } else {
    extractPromise = extractPlainText(file);
  }

  const timeoutPromise = new Promise<never>((_, reject) =>
    setTimeout(
      () => reject(new Error("File parsing timed out (15s).")),
      PARSE_TIMEOUT_MS,
    ),
  );

  const text = await Promise.race([extractPromise, timeoutPromise]);

  if (!text || text.trim().length === 0) {
    throw new Error("No text could be extracted from the file.");
  }

  return text.length > MAX_EXTRACTED_TEXT
    ? text.slice(0, MAX_EXTRACTED_TEXT)
    : text;
}

// ── PDF extraction via pdfjs-dist ─────────────────────────────────────────────

async function extractPdfText(file: File): Promise<string> {
  // Dynamic import to keep pdfjs out of SSR / initial bundle
  const pdfjsLib = await import("pdfjs-dist");

  // Configure worker using CDN; safe to set multiple times (idempotent)
  if (typeof window !== "undefined") {
    const ver =
      (pdfjsLib as unknown as { version?: string }).version ?? "5.6.205";
    pdfjsLib.GlobalWorkerOptions.workerSrc = `https://cdn.jsdelivr.net/npm/pdfjs-dist@${ver}/build/pdf.worker.min.mjs`;
  }

  const buffer = await file.arrayBuffer();
  const pdf = await pdfjsLib.getDocument({ data: buffer }).promise;

  const pageCount = Math.min(pdf.numPages, MAX_PDF_PAGES);
  const pages: string[] = [];

  for (let i = 1; i <= pageCount; i++) {
    const page = await pdf.getPage(i);
    const content = await page.getTextContent();
    const pageText = (content.items as Array<{ str?: string }>)
      .filter((item) => typeof item.str === "string" && item.str.trim() !== "")
      .map((item) => item.str as string)
      .join(" ");
    if (pageText.trim()) pages.push(pageText);
  }

  return pages.join("\n\n");
}

// ── DOCX extraction via JSZip (BUG-020 fix: proper ZIP/deflate decompression) ─

async function extractDocxText(file: File): Promise<string> {
  // Dynamic import: JSZip handles ZIP/deflate natively in browser & Node
  const JSZipModule = await import("jszip");
  const JSZip =
    (JSZipModule as unknown as { default?: typeof JSZipModule }).default ??
    JSZipModule;

  const buffer = await file.arrayBuffer();
  // JSZip.loadAsync handles deflate decompression correctly (fixes BUG-020)
  const zip = await (
    JSZip as unknown as {
      loadAsync: (
        b: ArrayBuffer,
      ) => Promise<{
        file: (
          name: string,
        ) => { async: (type: string) => Promise<string> } | null;
      }>;
    }
  ).loadAsync(buffer);

  const docFile = zip.file("word/document.xml");
  if (!docFile) {
    throw new Error("Invalid DOCX: word/document.xml not found.");
  }

  const xml = await docFile.async("string");

  // Extract readable text from WordprocessingML XML
  const text = xml
    .replace(/<w:p[ >\/]/g, "\n") // paragraph → newline
    .replace(/<w:br[^>]*\/>/g, "\n") // line break → newline
    .replace(/<w:tab\s*\/>/g, "\t") // tab character
    .replace(/<[^>]+>/g, "") // strip remaining XML tags
    .replace(/&amp;/g, "&")
    .replace(/&lt;/g, "<")
    .replace(/&gt;/g, ">")
    .replace(/&quot;/g, '"')
    .replace(/&#39;|&apos;/g, "'")
    .replace(/\n{3,}/g, "\n\n")
    .trim();

  return text;
}

// ── Plain text extraction ─────────────────────────────────────────────────────

function extractPlainText(file: File): Promise<string> {
  return new Promise<string>((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = (e) => {
      const result = e.target?.result;
      resolve(typeof result === "string" ? result : "");
    };
    reader.onerror = () => reject(new Error("Failed to read file."));
    reader.readAsText(file, "UTF-8");
  });
}
