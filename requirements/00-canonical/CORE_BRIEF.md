## Core Pillars

<!-- GUIDANCE: 3-5 non-negotiable principles that guide every decision. For each pillar, state the principle and explain what it means in practice. These are your "when two approaches conflict, this breaks the tie" rules.

Example pillars:
- **Data-Driven** — Every output grounded in real data, not generic advice
- **Targeted, Not Generic** — Personalized to the user's specific context
- **User Control** — User reviews and approves everything. Automation feels like a shortcut, not loss of agency
- **Speed** — Single-session value delivery. No step requires leaving the app
- **Privacy-First** — Data encrypted at rest, no third-party sharing, user can export/delete
-->

## Core Tensions

<!-- GUIDANCE: Every product has inherent tensions between competing goods. Name them explicitly. For each tension:

Common tensions:
- Automation vs Control — AI does the work but user stays in the loop
- Depth vs Speed — Thorough enough to be useful without feeling like a chore
- Free vs Paid — Free tier delivers real value without feeling crippled
